# AGENTS.md

## Purpose

This project performs exploratory data analysis, data quality assessment,
duplicate detection, organization name variant detection, cleaning
recommendation generation, dataset cleaning, and final reporting for a large
administrative Excel dataset.

All implementation must remain production-oriented, traceable, and safe for
repeated execution.

## Project Structure

The repository must follow this layout:

```text
14b-tp-eda-codex-nettoyage-donnees/
├── data/
├── reports/
├── src/
├── README.md
├── requirements.txt
├── AGENTS.md
└── PROJECT_PLAN.md
```

Expected runtime subfolders:

- `data/raw/`: original source files, never modified in place
- `data/processed/`: cleaned datasets and derived exports
- `reports/`: CSV and Markdown reports generated by the pipeline
- `src/`: runnable analysis scripts
- `src/utils/`: shared utilities for configuration, cleaning, similarity, and
  report generation

## Core Rules

1. Never overwrite the original dataset stored in `data/raw/`.
2. Inspect the workbook before making assumptions about sheet names, schema, or
   candidate columns.
3. Preserve original columns exactly as they appear in the source dataset.
4. Store cleaned or normalized values in separate derived columns.
5. Apply automatic corrections only when the confidence rule explicitly allows
   it.
6. Leave ambiguous matches in a review state instead of forcing a correction.
7. Use chunk-based processing where practical to reduce memory pressure on large
   files.
8. Add logging, docstrings, type hints, and defensive error handling across the
   codebase.

## Pipeline Outputs

The project must generate the following artifacts:

- `reports/eda_report.md`
- `reports/missing_values_report.csv`
- `reports/duplicates_report.csv`
- `reports/organization_variants_report.csv`
- `reports/cleaning_mapping.csv`
- `reports/final_report.md`
- `data/processed/fichier_nettoye.csv`

## Script Responsibilities

- `src/01_inspect_file.py`: inspect workbook structure, identify the primary
  data sheet, profile columns, and generate the EDA report
- `src/02_missing_values.py`: analyze missingness and assign handling
  recommendations
- `src/03_detect_duplicates.py`: identify exact, identifier, and probable
  duplicates
- `src/04_detect_text_variants.py`: detect likely organization name variants
  using similarity scoring
- `src/05_clean_data.py`: generate mappings, apply accepted corrections, and
  export the cleaned dataset
- `src/06_final_report.py`: synthesize all prior outputs into a final Markdown
  report

## Cleaning Principles

- Normalize text using trimming, uppercase conversion, accent removal,
  punctuation cleanup, special-character cleanup, and whitespace compression.
- Use RapidFuzz thresholds exactly as defined in the project plan:
  - `95-100`: accepted
  - `80-94`: review
  - `<80`: rejected
- Preserve a clear audit trail between source values, cleaned values, status,
  and justification.

## Reporting Principles

- Markdown reports must be readable and structured for evaluation.
- CSV reports must preserve enough detail for manual review and downstream
  analysis.
- Final conclusions must distinguish automated actions from items requiring
  human validation.

## Operational Notes

- Prefer reusable utility modules over duplicated logic in scripts.
- Fail with explicit error messages when required input files or expected sheets
  are missing.
- Make logs useful for debugging large-file runs and partial failures.
- Keep the code compatible with standard command-line execution using Python 3.
